## Summary
The UYA C99 backend emits `sizeof(i16)` for `@size_of(i16)`, but generated C does not define `i16`, so host C compilation fails.

## Affected Tasks
实现 `encode <input.yuv> --width --height --out out.ivf`。

## Toolchain Command
`/media/winger/_dde_data/winger/uya/gui-uya/uya/bin/uya build .agent/toolchain-bugs/repros/20260604-001242-size-of-i16.uya -o build/size-of-i16-repro`

## Actual Error
`error: 'i16' undeclared (first use in this function)` from generated C containing `sizeof(i16)`.

## Expected Behavior
`@size_of(i16)` should lower to a valid C expression such as `sizeof(int16_t)` or a compile-time constant.

## Repro File
`.agent/toolchain-bugs/repros/20260604-001242-size-of-i16.uya`

## Repro Code
```uya
export extern fn main() i32 {
    return @size_of(i16) as i32;
}
```

## Notes
The same lowering also breaks `make build` through `src/vp8/common/scratch.uya`, where `@size_of(i16)` appears in scratch byte-size calculations.
