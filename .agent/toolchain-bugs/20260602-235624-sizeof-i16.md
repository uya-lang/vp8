## Summary

`@size_of(i16)` passes UYA type checking but generates C code containing `sizeof(i16)`, where `i16` is not a C type or typedef in the generated translation unit.

## Affected Tasks

实现 SoA macroblock info arrays。

## Toolchain Command

`/media/winger/_dde_data/winger/uya/gui-uya/uya/bin/uya test src/vp8_common_mb_info_test.uya`

## Actual Error

The generated C fails to compile with:

```text
error: 'i16' undeclared (first use in this function)
```

## Expected Behavior

`@size_of(i16)` should lower to a valid C expression such as `sizeof(int16_t)` or a target-correct integer constant.

## Repro File
`.agent/toolchain-bugs/repros/20260602-235624-sizeof-i16.uya`

## Repro Code

```uya
use libc.malloc;

fn bytes_for_i16() usize {
    return @size_of(i16);
}

fn main() i32 {
    const ptr: &byte = malloc(bytes_for_i16()) as &byte;
    if ptr == null {
        return 1;
    }
    return 0;
}
```

## Notes

The VP8 macroblock info implementation works around this by using `2` bytes for each `i16` motion-vector lane.
